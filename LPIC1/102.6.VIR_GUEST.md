# VIRTUALIZATION GUEST

VM concept.

Host: Like my MAC right now

GUEST: Like my Ubuntu inside of VirtualBox.

Hypervisor Software or Virtual Machine Manager (VMM): ex: VirtualBox. Usually 32-bit CPUs can't run a VMM on themselves. For supporting VMMs, we can check `vmx` or `svm` or `kvm` in `/proc/cpuinfo` or with `lscput` in flag part. If it has a `hypervisor` flag, it shows that the machine is being run inside of a VM (it's a guest).

Type 1 Hypervisor: Hypervisor is installed directly on Hardware (there is no host OS) and we can have several OS inside of Hypervisor. EX: KVM, Hyper-V, Xen.

Type 2 Hypervisor: Like what I did with MAC, VirtualBox and Ubuntu.

Cloning process (from an installed machine).

OVF: Open Virtualization Format: An standard format to move machine between hypervisors.

OVA: Open Virtualization Archive: Archiving all standards.

Templating: Master copies to initiate new machines.
